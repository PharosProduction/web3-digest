# Правила написания статей

Правила для создания и редактирования статей дайджеста (`src/content/blog/*.md`).

## Структура статьи

Файл `src/content/blog/YYYY-MM-DD-slug.md` с frontmatter:

```yaml
---
title: Title up to 60 characters, no period
description: One sentence up to 160 characters.
pubDate: 2026-04-23
cover: https://replicate.delivery/...
source: https://example.com/news
tags: [mica, regulation]
---
```

## Редполитика

### Тематика

Новости крипторынка, блокчейна и регулирования цифровых активов (MiCA, лицензирование, комплаенс). Инструменты и инфраструктура для разработчиков и продуктовых менеджеров в web3.

### Язык

- Контент статей — только профессиональный разговорный US-English (американский английский).

### Стиль

- Информационный стиль: факты, без маркетинговой лексики, оценочных прилагательных и канцелярита.
- Предложения до 25 слов. Точки в конце буллетов.
- Без шаблона «от X до Y» / «from X to Y». Перечисляем конкретно.
- Без AI-маркеров: "delve", "landscape", "ever-evolving", "testament to", "tapestry", "seamless", "leverage", "in the realm of".

### Формат

- Заголовок: до 60 символов, без точки.
- Описание: одно предложение до 160 символов.
- Тело: 300–500 слов, 2–4 абзаца.
- Источник: обязательная ссылка на первоисточник.
- Обложка: URL от Replicate, соотношение 16:9.

### Обязательные поля frontmatter

`title`, `description`, `pubDate`, `cover`, `source` — все заполнены. Stop-хук (фаза 06) блокирует публикацию, если хоть одно поле пустое.
